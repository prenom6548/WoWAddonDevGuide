### Best Practices
1. ✅ **Start small** - Begin with a simple addon
2. ✅ **Study examples** - Read Blizzard's code in [07_Blizzard_UI_Examples.md](07_Blizzard_UI_Examples.md)

5. ✅ **Test frequently** - Use `/reload` often during development

### Common Mistakes to Avoid
1. ❌ Don't pollute global namespace
2. ❌ Don't use OnUpdate for everything (use events/timers)
3. ❌ Don't forget to validate saved variables
4. ❌ Don't create frames every time (use pools)
5. ❌ Don't hardcode strings (use localization)

### Critical 12.0.x Changes

**"Addon Apocalypse" - Secret Values System:**
- Combat-sensitive values (damage, healing, resources) are now "secret values"
- Addons cannot directly read these values; must use Blizzard's official damage meter
- Affects: damage meters, combat logs, healing trackers, threat meters
- C_DamageMeter namespace provides official encounter data

**Major API Migrations:**
- Action bar APIs moved to `C_ActionBar` namespace
- Combat log APIs moved to `C_CombatLog` namespace
- Many global functions removed (use C_* equivalents)

**New Systems:**
- Official damage meter and encounter timeline (`C_DamageMeter`)
- Housing system (`C_Housing`) - see [11_Housing_System_Guide.md](11_Housing_System_Guide.md)

**New TOC Directives:**
- `## Category:` - Addon category for organization
- `## Group:` - Group related addons together
- `## LoadSavedVariablesFirst:` - Ensure saved variables load before code

### 12.0.5 Additions (Current Patch)

**Secret-Safe Rendering:**
- New numeric formatter types: `AbbreviatedNumberFormatter`, `NumericRuleFormatter`, `SecondsFormatter` — accept secret numbers natively via `Format` on duration objects
- New `Cooldown:SetCountdownFormatter(formatter)` and `Cooldown:SetCountdownMillisecondsThreshold(seconds)` — reduces the need for `pcall(string.format)` workarounds

**Nameplates:**
- Per-plate hit-rect override API: `CanChangeHitTestPoints`, `SetHitTestPoints`, `SetAllHitTestPoints`, `ClearAllHitTestPoints`, `GetHitTestPoints` — replaces the global `C_NamePlate.SetNamePlateSize` trick for per-plate sizing

**Auras:**
- `isHelpful`, `isHarmful`, `isRaid`, `isNameplateOnly`, `isFromPlayerOrPlayerPet` are no longer secret — addons can classify auras via Lua during combat
- `auraInstanceID` values re-randomize at encounter / M+ / PvP boundaries

**Unit Identity Restrictions:**
- `UnitName`, `UnitSpellTargetName`, `UnitTokenFromGUID`, `UnitIsUnit` stricter about secret unit tokens — see [12_API_Migration_Guide.md](12_API_Migration_Guide.md) for comparison rules

**Miscellaneous:**
- `table.freeze(t)` / `table.isfrozen(t)` — read-only tables
- New `"outfit"` action type for `SecureActionButtonTemplate`
- `GetRaidRosterInfo` returns `"Unknown"` instead of `nil` for uncached names
- `UNIT_CONNECTION` event reliability fixed
- `IsInInstance` now correctly returns `true` inside delves
- Throttles relaxed during `PLAYER_LOGOUT` / `ADDONS_UNLOADING`
- New `Predicates` table in generated API docs describes per-API secret/taint restrictions
